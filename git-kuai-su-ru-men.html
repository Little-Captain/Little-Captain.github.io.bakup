Git vs SVN

系统的架构

SVN : 集中式

Git : 分布式

分布式架构的好处, 近乎所有的操作都是本地操作. 具体体现在: 1. 离线可以工作; 2. 操作速度快

对待数据的方式

SVN : 差异比较

Git : 文件快照

Git 中文件的三种状态

  • 已提交(committed) : 数据已安全的保存在本地数据库中
  • 已修改(modified) : 修改了文件, 还没保存在数据库中
  • 已暂存(staged) : 对一个已修改的文件的当前版本做了标记, 使之包含在下次提交的快照中
